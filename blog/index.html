Create an Interactive Console for Your Library With Pry

When working on a library written in Ruby the interactive part of trying and using it during development is sometimes hard and difficult. Tools like irb and Pry come in handy here. In general every RubyGem should have a console to have your library right at your fingertips. This makes learning the API and become comfortable with the interface way easier if someone wants to use your code or contribute to it.

The easiest way to do this with Pry is to push your models into the load path, require them and start a Pry command line.

In combination with pry-rescue and pry-debugger or pry-byebug this turns into a very powerful tool to speedup your development process. Within the console predefined structures and objects can be created to DRY up your workflow.
